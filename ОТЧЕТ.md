Отчет содержит описание разработки веб‑приложения формата SPA (single page application) «Магазин комплектующих для ПК». Реализованы страницы: главная, каталог, о нас, контакты, корзина. ## Термины и определения

**SPA (Single Page Application)** — веб‑приложение, в котором переход между разделами происходит без полной перезагрузки страницы.

**Роутер (маршрутизатор)** — модуль, отвечающий за отображение нужной «страницы» внутри SPA при навигации.

**localStorage** — встроенное хранилище браузера для долговременного хранения данных на стороне клиента.

**sessionStorage** — встроенное хранилище браузера для хранения данных в рамках текущей вкладки/сессии.

## 1 Аналитическая часть

### 1.1 Описание предметной области
Интернет‑магазин комплектующих для ПК предоставляет пользователю возможность просматривать товары по категориям (процессоры, видеокарты и т.д.), получать подробную информацию о товаре и добавлять товары в корзину.

### 1.2 Требования к системе

**Функциональные требования:**
- отображение списка товаров (каталог);
- поиск по названию/описанию товара;
- фильтрация по категории;
- сортировка по цене и рейтингу;
- постраничный вывод (пагинация);
- просмотр подробной информации о товаре (модальное окно);
- работа с корзиной (добавление, удаление, изменение количества);
- сохранение корзины между перезагрузками страницы.

**Нефункциональные требования:**
- адаптивная верстка;
- работа в современных браузерах;
- модульная структура кода и читаемость.

### 1.3 Выбор технологий
Выбран стек: HTML/CSS и JavaScript (ES‑модули). Для удобной разработки и сборки применяется Vite.

## 2 Проектирование

### 2.1 Архитектура приложения
Приложение реализовано в виде набора модулей:
- модуль роутинга (SPA‑маршрутизатор);
- модуль корзины и работы с хранилищем;
- UI‑компоненты страниц.

Каждая страница реализуется как UI‑компонент, который возвращает объект с DOM‑элементом и методом инициализации (при необходимости).

### 2.2 Навигация (роутинг)
Маршруты приложения:
- `home` — главная страница;
- `catalog` — каталог товаров;
- `about` — информация о проекте/магазине;
- `contacts` — контакты;
- `cart` — корзина.

Роутер хранит таблицу маршрутов и заменяет содержимое контейнера `[data-router-outlet]` при навигации.

### 2.3 Модель данных

**Товар** описывается полями:
- идентификатор;
- наименование;
- цена;
- категория;
- рейтинг;
- описание;
- ссылка на изображение.

**Корзина** хранит пары *(id товара → {товар, количество})*.

### 2.4 Хранение состояния
- **Корзина:** `localStorage` (ключ `mirea-shop-cart-v1`).
- **Выбор категории с главной страницы:** `sessionStorage` (ключ `selectedCategory`).

## 3 Реализация

### 3.1 Структура проекта

Основные файлы (актуально для репозитория):
- `src/main.js` — инициализация приложения, регистрация маршрутов, подключение компонентов;
- `src/modules/router.js` — роутер;
- `src/modules/cart.js` — корзина (Map, сохранение/загрузка из localStorage);
- `src/data/products.js` — список товаров (константа `PC_PRODUCTS`);
- `src/ui/` — UI‑компоненты страниц;
- `src/styles/main.css` — стили.

### 3.2 Реализация каталога
В каталоге реализованы:
- поиск по названию и описанию;
- фильтр по категориям;
- сортировка (по цене и по рейтингу);
- пагинация (лимит 12 товаров на страницу);
- открытие карточки товара в модальном окне;
- добавление товара в корзину.

### 3.3 Реализация корзины
Корзина реализована как отдельная страница. Доступные действия:
- просмотр списка выбранных товаров;
- изменение количества товара кнопками `+`/`−`;
- удаление товара;
- очистка корзины;
- расчет итоговой суммы.

Для демонстрации предусмотрена кнопка «Оформить заказ» (демо‑режим).

### 3.4 Реализация UX и интерфейса
Интерфейс выполнен в едином стиле, поддерживает адаптивность, использует сетки (Grid/Flex) и семантическую разметку.

## 4 Тестирование

Проверены основные пользовательские сценарии:

1. Переходы по разделам через шапку сайта.
2. Выбор категории на главной странице → переход в каталог с примененным фильтром.
3. Поиск товара по названию/описанию.
4. Сортировка товаров по цене и рейтингу.
5. Пагинация каталога.
6. Открытие модального окна товара.
7. Добавление товара в корзину.
8. Изменение количества в корзине, удаление товара.
9. Сохранение корзины при перезагрузке страницы (localStorage).

**Результат:** перечисленные сценарии выполняются корректно.
